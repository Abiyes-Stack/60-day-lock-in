## The operating thesis

This is not a certification sprint and not a portfolio-for-its-own-sake sprint.

The working loop is:

**fresh job posting → business-problem decode → proof-of-work sprint → tailored application + warm outreach → follow-up → interview evidence**

Projects and certifications exist to improve job conversion. Everything else is secondary.

### Job decision rules

Score every candidate role 0–2 on each:

1. Compensation and location are workable.
2. At least 60–70% of core requirements are credible.
3. Overlap with {{YOUR_DOMAINS — e.g. "accounting, systems, data, operations"}}.
4. A real referral or contact path exists ({{YOUR_STRONGEST_NETWORK}} = automatic 2).
5. Existing experience or a project can prove a relevant capability.

**8–10:** apply this week + seek warm referral. **6–7:** apply only if it supports a current project cluster. **<6:** skip entirely.

## Career conversion system

Run the pipeline in this order every cycle (see `pipeline/README.md`):

1. **Job scout** — scan target boards + market; score; pick the anchor. First seven days: score at least 20 fresh roles so later decisions are evidence-based.
2. **JD decode** — anchor → business problems, verbatim keywords, KPIs, cluster of ≥3 similar roles.
3. **Project derive** — cluster → one sprint spec with public data and one business question.
4. **Application pack** — per role: tailored summary + bullets, project link, warm outreach, follow-up date.

The tracker (`pipeline/tracker.csv`) is the source of truth. Respond to recruiters, referrals, and interview invitations within 24 hours.
